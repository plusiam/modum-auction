# modum-auction

모둠 해결책 경매 토의 웹학습지입니다.

사회자 모드와 개인 모드로 동작하는 실시간 웹학습지입니다.

- 사회자 모드: 방 생성, 단계 전환, 참여자 잠금/제한, 최종 모둠 해결책 정리
- 개인 모드: 개인 해결책 작성, 경매 점수 제출, 1등 해결책 보완 의견 입력
- 결과 내보내기: 최종 결과 카드를 PNG/PDF로 저장
- 연결 방식: Firebase Firestore + 익명 인증 또는 로컬 데모 저장소

## 실행

이 프로젝트는 빌드 없이 실행됩니다.

1. 터미널에서 현재 폴더로 이동합니다.
2. `npm start` 또는 `node server.mjs`
3. 브라우저에서 `http://localhost:4173`

## Firebase 연결

처음에는 `firebase-config.js`가 데모 모드로 설정되어 있습니다.

1. Firebase 콘솔에서 프로젝트를 만듭니다.
2. `Authentication`에서 `익명 로그인`을 켭니다.
3. `Firestore Database`를 생성합니다.
4. [`firebase-config.js`](/Users/yeohanki/Documents/Inbox/modum-auction/firebase-config.js)를 실제 프로젝트 값으로 수정합니다.
5. `window.APP_USE_DEMO = false;`로 바꿉니다.
6. [`firestore.rules`](/Users/yeohanki/Documents/Inbox/modum-auction/firestore.rules)를 Firestore 규칙에 적용합니다.

`firebase-config.example.js`는 예시 템플릿입니다.

## 데이터 흐름

- `rooms` 컬렉션: 방 제목, 질문, 현재 단계, 최종 결과, 사회자 UID, 모둠 정원(`maxMembers`), 현재 사용 좌석 수(`memberCount`)
- `rooms/{roomId}/members` 서브컬렉션: 참여자 이름, 해결책, 점수, 보완 의견, 접속 시각, 잠금/제한 상태
- 실시간 반영: Firestore `onSnapshot()` 또는 데모 모드 `localStorage + BroadcastChannel`

## 사용 흐름

1. 학생 사회자가 방을 만들고 모둠 정원(사회자 포함 2명~10명)을 정한 뒤 6자리 코드를 공유합니다.
2. 참여자가 코드로 입장해 개인 해결책을 작성합니다.
3. 사회자가 `2단계 경매 점수`로 전환하면 참여자가 가장 설득력 있는 해결책에 점수를 줍니다.
4. 사회자가 집계 결과를 보고 1등 해결책을 고릅니다.
5. `3단계 보완 토의`에서 참여자가 보완 의견을 제출합니다.
6. 사회자가 최종 모둠 해결책을 정리하고 이미지/PDF로 저장합니다.

## 확인할 점

- PNG/PDF 저장은 `html2canvas`, `jsPDF` CDN 로딩이 필요합니다.
- Firebase를 쓰면 배포 도메인을 Firebase 콘솔의 허용 도메인에 추가해야 합니다.
- Firestore 보안 규칙은 강화되었지만, 프로덕션 배포 전에는 교사 계정 정책에 맞게 추가 검토가 필요합니다.
- 최근 방 목록과 좌석 스트립은 `memberCount`를 기준으로 정원을 표시합니다. 참여 제한을 걸면 사용 좌석 수에서도 제외됩니다.
- 재입장: 동일 브라우저 탭에서는 기존 세션이 유지되며, 다른 탭/기기에서는 동일 UID로 재입장하면 기존 데이터가 병합됩니다.

## 보안 강화 사항 (2026-03-08)

### XSS 방어
- 모든 사용자 입력 값에 `escapeHtml()` 적용
- 문자열, 숫자 모든 동적 값을 이스케이프 처리하여 일관성 확보

### Firestore Rules 보강
- **문자열 길이 제한**: 제목 100자, 질문 500자, 내용 1000자, 상황 설명 2000자
- **memberCount 조작 방지**: 사회자도 ±1 변경만 허용, blocked 상태 변경과 동시에만 가능
- **필수 필드 검증**: phase, role 등 허용된 값만 저장 가능
- **권한 분리**: 본인은 id/role/blocked 변경 불가, 사회자는 id/role 변경 불가

### 에러 경계 추가
- export 함수에 try-catch 추가하여 실패 시 명확한 메시지 제공
- 모든 async 함수 에러가 UI로 전달되어 사용자에게 피드백

## 운영 전 검증 순서

실제 수업에 쓰기 전에는 아래 순서로 확인하는 편이 안전합니다.

1. `2명 시나리오`
   학생 사회자 1명과 참여 학생 1명이 입장해 해결책 작성, 단계 전환, 점수 입력, 최종안 저장이 모두 되는지 확인합니다.
2. `6명 시나리오`
   여러 학생이 동시에 입장해 좌석 스트립, 제출 수, 점수 집계, 1등 해결책 선택이 실시간으로 맞는지 확인합니다.
3. `10명 정원 시나리오`
   정원을 10명으로 만든 뒤 10번째까지는 입장되고 11번째는 차단되는지 확인합니다.
4. `참여 제한 시나리오`
   사회자가 특정 학생을 `참여 제한`으로 바꾸면 좌석 수가 즉시 줄고, 다시 허용하면 재입장 가능한지 확인합니다.
5. `재입장 시나리오`
   같은 브라우저 탭 새로고침 후 기존 방으로 복귀되는지, 다른 브라우저/기기에서는 새 참여자로 정상 입장되는지 확인합니다.
6. `내보내기 시나리오`
   최종 결과 카드가 PNG와 PDF로 저장되고 한글이 깨지지 않는지 확인합니다.
